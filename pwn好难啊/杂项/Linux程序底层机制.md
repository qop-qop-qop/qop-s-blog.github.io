
# stdin/stdout/stderr
这是三个程序数据传输的通道
- 标准输入(stdin) : fd=0,程序读数据的通道,来源可以是键盘,网络(nc连接),管道等,程序统一通过read(1,...)读取
- 标准输出(stdout) : fd=1,程序输出数据的通道,输出可到显示器,网络,文件,程序,通过weite(1,...)或者printf/puts输出
- 标准错误(stderr) : fd=2,程序输出数据的通道,默认用于输出错误/警告/崩溃信息,与stdout是两个独立的通道,即使stdout被屏蔽,往stderr输出(write(2,...))仍能被接收.

stdout,stdin,stderr本质---------FILE结构体指针
FILE类型:
C标准库(<stdio.h>)定义的结构体类型,可理解为"流的操作手柄",内部存储了流的核心信息(fd,缓冲区地址,缓冲模式,读写状态等).
属性:
- 是C标准库提供的全局变量,类型为FILE*(指针),存在于程序的.data或.bss段,有固定的内存地址.可查,可改.可替换.
- ::stdout(c++) :
作用域解析运算符::,表示从全局命名空间中获取stdout,避免与局部命名变量混淆,同理有::stdin,::stderr.
- stdout等指针指向的地址不是硬件的真实地址,而是操作系统虚拟出来的虚拟文件入口地址(VFS虚拟文件系统的入口标识);程序通过该地址告诉操作系统要往stdout写数据,由内核负责把数据转发到底层硬件(显示器,网卡等),程序不直接操作硬件

## ID权限问题
每个进程开启会分别记录六个ID
- gid (组ID)
rgid  记录开启进程的组身份
egid  记录运行进程的组权限
sgid  备份初始egid,可修改,主要是帮助普通组临时回到高权限
- uid
ruid
euid
suid    作用参考gid

函数举例:
- getegid()获取当前进程的egid
- setresgid(,,)分别设置rgid egid sgid为参数值


# 虚拟地址空间(以Linux为例)

现代操作系统中,每个进程都有一个独立,连续的虚拟地址空间,而虚拟地址空间
是按页(4k大小)与物理内存一一对应,按需分配.用户态的程序基本只能接触虚拟地址空间,
堆内存的操作也基本是对虚拟空间的操作.

- kernel space  系统内核的内存映射
- stack 进程的栈内存
- dynamic link libraries  动态链接库的内存映射
- heap  为堆内存
- ELF  可执行程序的内存映射
- GAP  未使用的空白内存
